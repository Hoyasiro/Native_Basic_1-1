페르소나와 시스템 프롬프트가 결과 품질(일관성/안전성/형식 준수)에 미치는 영향을 설명할 수 있다.
생성 결과를 테스트로 검증하고(특히 사실 오류), 프롬프트를 반복 개선하는 과정을 재현 가능하게 설명할 수 있다.

## 1. 선정 과업: 메일 답장 초안 작성 (불만, 정보제공, 조언 등)
> 과업 선정 사유
  1. 실무밀착: 실제 매일 소요가 있는 일
  2. 환각검증: 없는 내용을 지어내는 지에 대한 검증이 명확
  3. 문맥유지: 10턴 대화 동안 원본 메일의 맥락과 조건 변경 유지 여부 파악에 용이
  4. 모델차별화: 맥락 독해력 차이가 모델마다 확연히 차이남
  5. 확장성: 지인 / 업무용 / 팀원 메일 등 다양한 변형 가능

계약서 발송 주소 확인은 상품명 VX의 시판 후 조사 신규 계약에 대해서만 메일을 주셨고, 동봉된 내용물은 OS 변경계약서와 시판 후조사 신규계약서 두 과제가 한 번에 왔습니다.
심지어 L파일이나 클립 등으로 구분되어 있지 않고 포스트잇으로 설명도 없으니 받은 사람 입장에서 열어보면 무엇인지 알 수 없었습니다.
